# 上线检查清单

## 一、环境准备

- [ ] 服务器已安装 Docker 与 Docker Compose
- [ ] 域名已解析到服务器
- [ ] 防火墙已开放 `80` / `443` / 必要 SSH 端口
- [ ] 已准备正式 `.env` 文件
- [ ] `POSTGRES_PASSWORD` 已替换为强密码
- [ ] `CORS_ORIGINS` 已改为正式域名
- [ ] `VITE_API_BASE_URL` 已改为正式 API 域名

## 二、部署前检查

- [ ] 仓库代码已拉取到最新 `main`
- [ ] 执行过 `./scripts/backup-db.sh` 备份旧数据
- [ ] `uploads/` 已配置持久化目录或备份策略
- [ ] `deploy/backup/` 目录具备写权限
- [ ] Nginx / 网关反向代理配置已准备完成

## 三、启动检查

- [ ] 执行 `./scripts/start.sh`
- [ ] `docker compose ps` 显示全部服务 `Up`
- [ ] `http://localhost:3000/health` 返回成功
- [ ] 后台页面可打开
- [ ] 移动端页面可打开
- [ ] PostgreSQL 容器日志无异常

## 四、业务验证

- [ ] 管理员登录成功
- [ ] 调度登录成功
- [ ] 现场账号登录成功
- [ ] 可创建项目
- [ ] 可按模板生成工单
- [ ] 可创建搭建清单
- [ ] 可导入清单 CSV
- [ ] 可指派工单与审核人
- [ ] 现场端可开始执行并提交审核
- [ ] 审核人可通过 / 驳回工单
- [ ] 可上传 / 删除附件
- [ ] 审计日志可查看

## 五、上线后运维

- [ ] 设置数据库定时备份
- [ ] 设置 `uploads/` 定时备份
- [ ] 配置日志采集或保留策略
- [ ] 建立发布回滚流程
- [ ] 记录当前发布版本号与时间
