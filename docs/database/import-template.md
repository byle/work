# 搭建清单 Excel 导入模板规范

## 1. 文档信息

- 项目名称：舞台租赁工单系统
- 文档版本：V0.1
- 文档日期：2026-03-25
- 适用范围：项目搭建清单导入

## 2. 目标说明

本规范用于定义“搭建清单 Excel 导入模板”的标准格式，统一现场清单录入方式，支持将已有 Excel 清单快速导入系统。

适用场景：

- 项目经理将现有搭建清单导入系统
- 内勤根据标准表格批量录入项目清单
- 后续模板化项目快速生成基础清单

## 3. 设计原则

- 尽量贴合当前线下使用的表格习惯
- 兼顾系统结构化存储要求
- 降低现场和内勤人员学习成本
- 支持表头信息 + 明细信息同时导入
- 支持后续扩展分类、备注、执行状态等字段

## 4. 推荐文件格式

支持以下文件格式：

- `.xlsx`（推荐）
- `.xls`（兼容）

建议限制：

- 单文件大小不超过 `10MB`
- 单次导入明细不超过 `2000` 行

## 5. Excel 模板结构

建议 Excel 分为两个区域：

- 表头区域：项目基础信息
- 明细区域：搭建清单项目明细

建议使用一个工作表完成导入，工作表名称建议为：

- `搭建清单`

## 6. 表头区域规范

表头区域建议放在前几行，格式如下：

| 字段名称 | 示例值 | 是否必填 | 说明 |
|---|---|---|---|
| 项目名称 | 第三届“新突围”所长沙龙 | 是 | 对应项目名称 |
| 项目地点 | 广西·北海·喜来登度假酒店-多功能厅 | 是 | 项目执行地点 |
| 活动时间 | 2026-03-26 | 是 | 活动日期 |
| 进场时间 | 2026-03-25 14:00 | 否 | 允许精确到分钟 |
| 撤场时间 | 2026-03-26 21:00 | 否 | 允许精确到分钟 |
| 彩排时间 | 2026-03-25 17:00 | 否 | 允许精确到分钟 |
| 备注 | VIP 活动，注意舞台边封 | 否 | 项目整体备注 |

### 表头填写要求

- 字段名称必须与模板定义一致
- 日期建议统一格式为：`YYYY-MM-DD`
- 日期时间建议统一格式为：`YYYY-MM-DD HH:mm`
- 如果系统中项目已存在，可支持按项目名称匹配或由用户导入时选择项目

## 7. 明细区域规范

明细区域建议从固定标题行开始，列头如下：

| 列名 | 是否必填 | 示例 | 说明 |
|---|---|---|---|
| 序号 | 否 | 1.1.1 | 支持层级编号 |
| 分类 | 否 | 外场物料清单 | 用于分组展示 |
| 内容 | 是 | 签到处背景板 | 搭建项名称 |
| 规格 | 否 | L5*H3m U60cm | 规格说明 |
| 数量 | 是 | 1 | 支持整数或小数 |
| 单位 | 是 | 套 | 单位 |
| 备注 | 否 | 多功能厅门口左边 | 特殊说明 |

### 明细字段说明

#### 1）序号

- 可为空
- 支持格式如：`1`、`1.1`、`1.1.1`
- 用于保留原始清单层级感，不作为唯一标识

#### 2）分类

- 可为空
- 用于区分清单分组，例如：
  - 外场物料清单
  - 大屏&舞台设备
  - 灯光设备
  - 音响设备
- 若为空，系统可默认归类为“未分类”

#### 3）内容

- 必填
- 表示具体搭建项名称
- 长度建议不超过 `200` 字符

#### 4）规格

- 选填
- 用于描述尺寸、型号、工艺等
- 长度建议不超过 `255` 字符

#### 5）数量

- 必填
- 支持正整数或小数
- 不允许为负数
- 建议最大值不超过 `999999.99`

#### 6）单位

- 必填
- 示例：`套`、`个`、`平方`、`米`
- 长度建议不超过 `20` 字符

#### 7）备注

- 选填
- 用于记录摆放位置、特殊要求、执行提醒
- 长度建议不超过 `255` 字符

## 9. 导入校验规则

## 9.1 表头校验

- 项目名称不能为空
- 项目地点不能为空
- 活动时间不能为空
- 日期格式必须合法
- 若存在进场/撤场/彩排时间，必须为合法日期时间格式

## 9.2 明细校验

- 内容不能为空
- 数量不能为空
- 数量必须为数字且大于 `0`
- 单位不能为空
- 单行字段长度不得超过系统限制

## 9.3 文件校验

- 文件格式必须为 `.xlsx` 或 `.xls`
- 工作表不能为空
- 明细标题行必须存在
- 明细数据至少有 `1` 行

## 10. 系统导入处理规则

### 10.1 项目处理规则

建议支持两种模式：

#### 模式 A：导入到已有项目

- 用户先在系统中选择项目
- Excel 中表头项目名称仅作为校验或补充展示
- 导入后生成该项目下的搭建清单

#### 模式 B：导入并自动创建项目

- 如果系统中不存在对应项目，可根据 Excel 表头自动创建项目
- 创建规则需结合权限控制和业务流程决定

MVP 阶段建议优先采用：

- **模式 A：导入到已有项目**

这样更稳定，也便于权限和数据管理。

### 10.2 清单处理规则

- 每次导入生成一份新的搭建清单记录
- 导入成功后写入导入记录
- 明细按照 Excel 行顺序生成排序字段
- 若分类为空，系统自动补为“未分类”
- 若序号为空，系统可自动生成内部排序值，但保留空显示逻辑

### 10.3 重复数据处理规则

MVP 建议采用：

- 不做复杂去重
- 每次导入视为一份新的清单版本
- 如需覆盖旧数据，由用户手工确认替换或归档旧清单

## 11. 导入结果反馈

导入完成后，系统应返回：

- 导入状态：成功 / 部分成功 / 失败
- 总条数
- 成功条数
- 失败条数
- 失败原因列表

推荐展示示例：

- 总计 `48` 条
- 成功 `46` 条
- 失败 `2` 条
- 失败原因：
  - 第 `12` 行数量格式错误
  - 第 `18` 行内容为空

## 12. 导入失败行处理建议

为了提升可用性，建议系统支持：

- 返回失败行号
- 返回失败字段
- 返回失败原因
- 支持用户修正后重新导入

- 查看导入说明
