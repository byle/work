# 舞台租赁工单系统 PRD

## 1. 文档信息

- 项目名称：舞台租赁工单系统
- 文档版本：V0.1
- 文档日期：2026-03-25
- 适用阶段：MVP 需求定义
- 目标端：`Web` 后台、手机 `H5`，后续扩展 `微信小程序`

## 2. 项目背景

舞台租赁业务通常围绕活动项目展开，涉及项目创建、设备与物料准备、现场搭建、彩排保障、活动执行、撤场回收等多个环节。当前很多团队仍依赖 Excel、微信群、电话和纸质清单协同，导致以下问题：

- 项目信息分散，项目经理难以统一管理
- 搭建清单依赖手工维护，版本混乱
- 工单分派和执行状态不透明
- 现场反馈不及时，异常难追踪
- 常用活动的搭建方案重复录入，效率低

因此，需要建设一套适合舞台租赁公司的工单系统，统一管理项目、工单、搭建清单和现场反馈流程。

## 3. 项目目标

### 3.1 业务目标

- 统一管理活动项目、搭建工单和撤场工单
- 提升项目创建、派工、执行反馈效率
- 将常见项目内容模板化，减少重复录入
- 支持搭建清单录入与 Excel 导入
- 为现场人员提供轻量化手机端操作入口

### 3.2 系统目标

- 提供项目级数据中心，集中查看项目全貌
- 提供模板化生成能力，支持快速创建项目与工单
- 提供现场任务执行能力，支持勾选、备注、拍照反馈
- 支持后续扩展到设备、仓库、排期、统计分析模块

## 4. 用户角色

### 4.1 超级管理员

负责系统初始化配置、账号管理、角色权限配置。

### 4.2 项目经理

负责项目创建、搭建清单维护、工单生成与整体进度跟进。

### 4.3 调度/内勤

负责工单分派、时间协调、项目安排与进撤场协同。

### 4.4 现场负责人

负责现场执行推进、异常处理、完工确认。

### 4.5 搭建人员

通过手机端查看任务、执行任务并反馈状态。

### 4.6 仓库/物料人员

负责备料、设备出入库确认，后续可扩展与清单联动。

## 5. 核心业务场景

### 5.1 项目创建

项目经理在 Web 后台创建活动项目，填写：

- 项目名称
- 项目地点
- 活动时间
- 进场时间
- 撤场时间
- 彩排时间
- 客户信息
- 项目负责人
- 备注

### 5.2 工单生成与分派

系统支持基于项目手工创建工单，或从模板批量生成工单，例如：

- 搭建工单
- 运输工单
- 撤场工单
- 补料工单
- 临时加单

创建后可分配执行人，并跟踪执行状态。

### 5.3 搭建清单录入与导入

项目经理可：

- 手工录入搭建清单
- 从 Excel 导入清单
- 维护清单分类、内容、规格、数量、单位、备注
- 将清单作为现场执行依据下发给手机端人员

### 5.4 现场执行反馈

现场人员通过手机端：

- 查看待处理项目与工单
- 查看搭建清单明细
- 勾选完成项
- 填写备注和异常说明
- 上传现场照片
- 提交完成结果

## 6. 产品范围

### 6.1 MVP 范围

#### 后台管理端

- 登录
- 用户与角色基础管理
- 项目列表、项目创建、项目详情
- 工单列表、工单创建、工单分配、工单状态流转
- 搭建清单列表、录入、编辑、导入
- 模板基础能力（项目模板 / 工单模板）
- 图片查看与执行记录查看

#### 手机端 H5

- 登录
- 我的项目
- 我的工单
- 项目清单查看
- 工单执行
- 勾选完成
- 备注反馈
- 图片上传

### 6.2 非 MVP 范围

以下能力首期不进入 MVP：

- 财务结算
- 合同管理
- 复杂审批流
- 多仓库联动
- 设备 RFID / IoT
- 小程序正式版
- 智能排班与资源冲突预警

## 7. 功能需求

## 7.1 账号与权限

### 功能描述

系统支持账号登录和基于角色的权限控制。

### 需求点

- 支持账号密码登录
- 支持角色区分：管理员、项目经理、调度、现场负责人、搭建人员
- 不同角色看到不同菜单与数据范围

### MVP 验收标准

- 管理员可创建账号
- 用户登录后仅能看到授权页面
- 现场人员只能查看分配给自己的任务

## 7.2 项目管理

### 功能描述

以项目作为主业务对象，承载工单、清单、附件和日志。

### 需求点

- 项目列表查询
- 创建项目
- 编辑项目
- 查看项目详情
- 记录关键时间节点
- 绑定负责人和参与人员

### 核心字段

- 项目编号
- 项目名称
- 客户名称
- 项目地点
- 活动时间
- 进场时间
- 撤场时间
- 彩排时间
- 项目状态
- 项目负责人
- 备注

### MVP 验收标准

- 后台可新增项目并保存完整信息
- 可在项目详情页看到关联工单和清单

## 7.3 工单管理

### 功能描述

围绕项目执行任务建立工单体系。

### 需求点

- 工单创建
- 工单类型管理
- 工单分配执行人
- 工单状态流转
- 工单执行备注
- 工单附件上传
- 工单操作日志

### 工单类型建议

- 搭建工单
- 撤场工单
- 运输工单
- 补料工单
- 异常处理工单

### 工单状态建议

- 待分配
- 待执行
- 执行中
- 已完成
- 已关闭

### MVP 验收标准

- 项目下可以创建多个工单
- 工单可分配到具体人员
- 现场人员可在手机端提交完成结果

## 7.4 模板管理

### 功能描述

将常见活动方案沉淀为模板，提升项目录入效率。

### 需求点

- 创建项目模板
- 创建工单模板
- 模板可配置默认清单项
- 支持从模板一键生成项目和工单

### MVP 验收标准

- 可维护至少一种模板
- 可从模板生成项目搭建清单和工单

## 7.5 搭建清单管理

### 功能描述

系统支持与当前 Excel 表格类似的搭建清单结构化管理。

### 表头字段

- 项目名称
- 项目地点
- 活动时间
- 进场时间
- 撤场时间
- 彩排时间
- 备注

### 明细字段

- 序号
- 分类
- 内容
- 规格
- 数量
- 单位
- 备注
- 执行状态
- 执行人
- 完成时间

### 需求点

- 清单手工新增/编辑/删除
- 清单分组展示
- Excel 导入
- 按项目查看清单
- 手机端查看清单
- 清单项执行反馈

### MVP 验收标准

- 能成功导入标准格式 Excel 清单
- 导入后可在项目详情页查看
- 手机端可查看清单并回传执行状态

## 7.6 手机端执行

### 功能描述

手机端为现场人员提供最简操作路径。

### 页面建议

- 登录页
- 我的项目
- 我的工单
- 工单详情
- 搭建清单详情
- 异常反馈页

### 需求点

## 9. 业务流程

### 9.1 项目创建流程

1. 项目经理创建项目
2. 录入项目基本信息
3. 选择模板或手工录入清单
4. 生成工单
5. 分派执行人

### 9.2 现场执行流程

1. 执行人登录手机端
2. 查看待办工单
3. 打开工单详情
4. 查看搭建清单
5. 勾选完成项
6. 填写备注/异常
7. 上传照片
8. 提交完成

### 9.3 撤场流程

1. 项目结束后生成撤场工单
2. 分配人员执行撤场
3. 现场确认回收完成
4. 提交撤场结果

## 11. 技术与架构建议

### 11.1 客户端

- 后台：`Vue / React` 均可
- 手机端：优先 `H5`，兼容微信内打开

### 11.2 服务端

- RESTful API
- 文件上传服务
- Excel 导入解析能力
- 统一权限控制

### 11.3 数据库

- `MySQL` 或 `PostgreSQL`
- 首期更推荐 `PostgreSQL`

## 12. 风险与注意事项

- 清单导入模板必须尽早确定，否则后续导入规则易频繁变更
- 现场手机网络不稳定，需要页面尽量简洁，上传流程要稳
- 角色权限边界要尽早明确，避免后期返工
- 模板能力不要首期设计过重，先满足常用活动场景
